The game itself is based on a hex grid that will look at each cell neighborhood around it to determine its destiny. It will cycle through all six neighborhoods and add up all the values determined by the rule set. This cell will then look up its life/death status based on the number of neighborhood cell value.

The following image shows a rule set and all of its values for counting the neighborhood. The black is the rule lookup value, and the grey is this rule's counter value.

For the following sample region each of the neighborhoods are compared vs the rule and the values of the rule is totaled up. The sample region with our sample rule would have a value of 0 since each neighborhood had a value of 0 in the rule set.

The following shows the life/death status. The top is the birth status, and the bottom is the death. This table shows that birth will only happen if there is 3 neighbors. The cell will stay alive only if there are 2 or 3 neighbors. This rule is similar to the game of life.

Since a hex grid can be represented as a grid, all of the calculations on position are simplified by just using a 2D array. This makes calculation easier, but the display will not be complete until it displays a hex grid as well.
